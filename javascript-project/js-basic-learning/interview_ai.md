# AI

## AI 基本概念

- AI 应用开发流程(AI 应用核心在模型，模型需要数据训练，结果生成后调优)

  - 训练：训练是 AI 模型开发过程中关键的一个步骤

    - 数据收集和准备
    - 特征提取和工程
    - 模型选择
    - 训练过程
    - 验证和测试
    - 超参数调优

  - 模型：

    - 定义和结构：不同模型决定了它如何处理输入数据和生成不同的输出结果
    - 训练后的知识储备：经过训练的模型包含了从数据中学习到的模式和知识，当遇到新问题时，可以利用这些知识进行推理和决策。
    - 推理能力：训练好的模型能够对新的，未见过的数据进行推理，输出预测的结果
    - 常见模型：
      - GPT-3.5
      - GPT-4
      - GPT-4o
      - T5(文本到文本转换器)
      - Claude
      - 字节豆包

  - Prompt：

    - 定义：Prompt 是用户给模型提供的指令或提示，用于引导模型产生特定的输出，如：在自然语言处理中，prompt 可以是一个问题、一个不完整的句子等。
    - 设计：设计有效的 Prompt 是确保模型产生高质量输出的重要因素，好的 Prompt 应该明确、具体、并与模型训练和数据相关。
    - 应用场景：Prompt 用于各种 AI 任务中，如对话系统、文本生成、代码补全等；通过提供不同的 prompt,用户可以实现多种不同功能。

  - 输出：是 AI 模型根据输入的 Prompt 生成的结果
    - 输出形式：可以是多种形式，可以是文本、图像、音视频等；取决于具体的任务和模型的类型
    - 评价指标：评估输出质量方法包括：准确率、召回率、F1 分数等
    - 调优和改进：通过分析输出结果，可以发现模型的不足之处，从而针对性的调优和改进。如：重新训练模型、优化 Prompt 设计、增加数据量等方法

- AI 具体应用案例

  - 客服聊天机器人：开发自动化客服系统，提升客户服务效率
  - 内容生成工具：自动化内容营销工具，帮助企业生成营销文案、博客文章等
  - 数据分析和报告：开发基于 AI 的数据分析和报告生成工具、提升业务决策效率
  - 医疗行业：疾病预测和诊断，提高医疗效率和准确性
  - 金融行业：风险控制(开发 AI 模型进行风险预测和控制，提高金融产品安全性)；智能投顾(提供基于 AI 的投资建议，帮助用户进行理财决策)
  - 教育行业：智能教学助手(提供个性化学习方案和即时反馈)

- 未来需要了解的业务场景

  - 云表格
  - 多维表
  - 文档编译器
  - 数据可视化

- AI 基本概念名词
  - LLM(大语言模型)：一种基于深度学习的模型，能够理解和生成自然语言文本
  - 智能体(Agent)：能“感知环境 → 规划步骤 → 调用工具 → 执行动作 → 根据结果继续决策”的智能体
    - 环境(Environment)：智能体存在的外部世界，它提供了智能体进行决策和交互的环境
    - 状态(State)：智能体在环境中的一种特定配置或条件，它描述了智能体当前的情况
    - 动作(Action)：智能体在环境中执行的操作，它改变了智能体的状态
    - 奖励(Reward)：智能体在环境中执行动作后获得的反馈信号，它用于指导智能体学习和改进策略
    - 策略(Policy)：智能体根据当前状态选择动作的规则或函数，它决定了智能体的行为
